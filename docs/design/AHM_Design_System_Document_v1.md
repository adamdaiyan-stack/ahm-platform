```
AHM PLATFORM — DESIGN SYSTEM DOCUMENT
Version 1.0
UI/UX & Visual Standards

1. DESIGN PHILOSOPHY

The AHM Platform should feel:
- premium
- intelligent
- modern
- institutional
- minimal
- fast
- trustworthy

The interface should combine:
- Bloomberg-level seriousness
- Apple-level simplicity
- TradingView-level usability
- Institutional-grade hierarchy

The platform must never feel:
- cluttered
- overly retail
- cheap
- gamified
- visually noisy

2. CORE DESIGN PRINCIPLES

PRINCIPLE 1 — Intelligence First
The platform should prioritize insight over raw data.

PRINCIPLE 2 — Progressive Disclosure
Show users only what matters first.
Advanced detail should appear only when needed.

PRINCIPLE 3 — Clarity Over Density
Too much information reduces usability.
Whitespace is a feature.

PRINCIPLE 4 — Consistency
Every page should feel like part of one system.

PRINCIPLE 5 — Modular Design
All UI sections must be reusable and standardized.

3. DESIGN STYLE DIRECTION

The platform aesthetic should feel:
- dark modern finance terminal
- clean institutional dashboard
- AI-powered financial intelligence system

Primary inspirations:
- Bloomberg Terminal
- Koyfin
- TradingView
- Linear
- Stripe Dashboard
- Apple UI hierarchy

4. COLOR SYSTEM

```

```
PRIMARY BACKGROUND
- Deep charcoal / near-black

SECONDARY BACKGROUND
- Slightly elevated dark grey

ACCENT COLORS
- Green = positive
- Red = negative
- Blue = intelligence/system actions
- Amber = warnings
- Grey = neutral/inactive

RULES
- Avoid excessive saturation
- Avoid neon colors
- Use muted premium tones
- Colors must communicate meaning

5. TYPOGRAPHY SYSTEM

PRIMARY FONT STYLE
Modern sans-serif with institutional feel.

TYPOGRAPHY HIERARCHY

Heading 1
- Major page titles

Heading 2
- Section headers

Heading 3
- Card titles

Body Text
- Main readable content

Caption Text
- Secondary information

RULES
- Strong visual hierarchy
- Avoid excessive font weights
- Avoid too many font sizes
- Use spacing instead of decoration

6. SPACING SYSTEM

RULES
- Consistent spacing across all pages
- More whitespace than typical finance apps
- Avoid cramped layouts

STANDARDIZATION
- Small spacing
- Medium spacing
- Large spacing
- Section spacing

Whitespace should guide attention.

7. LAYOUT SYSTEM

```

```
STANDARD PAGE STRUCTURE

Page Header
↓
Primary Intelligence Section
↓
Secondary Data Sections
↓
Expandable Detailed Data
↓
Footer/Utilities

RULES
- One primary focus per screen
- Avoid overcrowding
- Avoid multiple competing highlights
- Sections must feel visually grouped

8. CARD SYSTEM

CARD TYPES

Primary Intelligence Cards
- Key insights
- Major market movement
- AI summaries

Metric Cards
- KPI values
- Market numbers
- Portfolio metrics

Data Cards
- Tables
- Charts
- Analytics

RULES
- Consistent padding
- Consistent border radius
- Consistent elevation
- No unnecessary shadows
- Avoid card overload

9. TABLE SYSTEM

TABLE PRINCIPLES
- Tables must be readable
- Reduce unnecessary columns
- Important information first
- Use whitespace generously

REQUIRED FEATURES
- sorting
- filtering
- pagination
- responsive layout
- loading state
- empty state

RULES
- One universal table component
- No custom tables unless approved

```

```
- All tables must use shared styling

10. CHART SYSTEM

CHART PRINCIPLES
- Charts must communicate quickly
- Simplicity over complexity
- Avoid visual clutter

STANDARD CHART TYPES
- Line charts
- Bar charts
- Area charts
- Heatmaps
- Candlestick charts

RULES
- Consistent axis styling
- Consistent tooltip styling
- Consistent gridline opacity
- Avoid excessive labels

11. ICONOGRAPHY

RULES
- Minimal icon usage
- Icons should support meaning
- Avoid decorative icon clutter

Preferred style:
- clean outline icons
- consistent stroke width

12. NAVIGATION SYSTEM

PRIMARY NAVIGATION
- Clean sidebar navigation
- Simple hierarchy
- No excessive nesting

NAVIGATION PRINCIPLES
- Users should always know where they are
- Navigation should feel effortless
- Important areas easily accessible

13. RESPONSIVE DESIGN PRINCIPLES

The platform must work across:
- desktop
- laptop
- tablet
- mobile

RULES
- Mobile-first awareness
- Important information prioritized
- Reduce complexity on smaller screens

14. LOADING STATES

REQUIRED
Every page/component must include:
- loading state
- error state

```
- empty state

RULES

15. DATA VISUALIZATION PRINCIPLES

RULES
- Highlight insight, not just numbers
- Prioritize trends over raw data
- Use annotations sparingly
- Avoid excessive indicators

16. AI/INTELLIGENCE UI PRINCIPLES

AI OUTPUTS SHOULD FEEL:
- concise
- insightful
- actionable
- institutional

EXAMPLES
GOOD:
“Cement sector rallied due to declining coal prices.”

BAD:
Large unnecessary AI paragraphs.

RULES
- AI summaries must be short
- Focus on why something matters
- Avoid overwhelming users

- premium

The platform should never feel:
- chaotic

- distracting
